# Modularitas: Fungsi (Return) dan Prosedur (Void)

**Ringkasan Materi (TL;DR)**
Dalam mencacah arsitektur program perangkat raksasa kompilator utuh, penulisan sintaks program secara linear lurus disebuah blok tunggal merongrong pemahaman pusing dan redudansi fatal berlipat (mengulangi blok kode di kopas salin berulang ulang boros sama persis). Revolusi konsep algoritma ini diselamatkan utilasi komputator rimpangan modul terpisah melalui metode perbaikan **Modularitas**. Sepotong blok tugas yang independen dikemas menjadi teritori sub-program (subrutin). Ada Sub-program parameter fungsi jenis pemutar rujukan utilitas nilai *(Return Function)* dan tipe pengemban eksekusi komando aksi sepihak mutlak murni rujukan fungsi tipe *(Void/Prosedur)*.

**Konsep Ruang / Arsitektur Struktur Pendelegasian Modul**
Pendeklarasian utiliti parameter modular dirajut dengan menyediakan *Parameter Inputan Argumen Referensi*: 
1. **Fungsi / Metode yang Mengembalikan Nilai (*Return Function*)**:
Pilar modul fungsi ini dirancang diforsir mensyaratkan kewajiban komputasi mengeksekusi perhitungan operasi pemrosesan kalkulasi presesi hasil utilitas rumit di dalamnya, lalu dipaksa divalidasi meluncurkan melempar menembakka peluru nilai mutlak final luaran keluaran (*Return Value*) presesinya utuh dilesatkan merujuk melayang rasion kembali merapat pada letak fungsi modul rimpang induk sistem pemanggilnya awal eksekutor utilasinya tadi (*Main function/Caller*). Modul diwajibkan menjanjikan penegas tipe nilai rujukan perbatasan lemparannya presisi rasio apa (Misal `int`, `float`). Contoh: Blok komputasi modul utilitas `Hitung_Luas_Persegi(panjang, lebar)` mereturn menembakkan nilai lemparan perpaduan luas integer 20 hasil komparasinya ke penampung.
2. **Prosedur / Metode Kosong Purna Aksi Hampa (*Void Function*)**:
Pilar modul ini dicanangkan utilasinya murni mendelegasi merancang sekedar serangkaian aktivitas "Tindakan Aksi Pelaksanaan Murni Eksekutor Pekerja Suruhan". Sehabis blok kode prosedurnya ini telah tuntas mentok merajut rimpang operasi batas limit aksinya dikerjakannya lurus (seperti menyalakan lampu menu, membuka file print layar UI), Fungsi ini dilarang tak dituntut mendelagasikan perlembaraan lemparan peluru nilai output jawaban nilai apapun untuk kembali ke asalnya. Format batasan pengikat pilar deklarasinya diawali instruksi memori batas rujukan nilai tipe eksak nihil murni hampa yakni parameter: **`void`**.

**Latihan Soal & Pembahasan (Format Q&A)**

**Q1**: Simak pseudo program percabangan pendelagasian modul fungsi berjejer dan prosedur pemanggil di bawah ini: 
```cpp
void Cetak_Header_Tagihan() {
    print("----- INVOICE TRANSAKSI ANDA -----");
}

int Hitung_Pajak(int bayar_mentah) {
    int total_pajak = bayar_mentah * 0.1;
    return total_pajak;
}
```
Uraikan bedah kenapa batas komputasi modul struktur sintaks awal parameter mutlak *Cetak_Header_Tagihan* merujuk memakai komputator utilasi `void`, disisi berlainan perbatasan blok instruksional komparasi utilasi susunan subrutin modul *Hitung_Pajak* menancapkan rujukan tipe nilai komputator presisi mutakhir `int`?
**A1**: Relasi ini merepresentasikan pembagian pemisah tugas rasional. Modul struktur *Cetak_Header_Tagihan* dikhususkan murni memanipulasi serangkaian porsi *action* utiliasi mencetak karakter layar belaka, yang mana batas operasi ini selesai tertutup lunas sendirinya pasca mencetak ke perbatasan batas tampilan UI terminal. Dia tidak memiliki utilasi parameter balasan data logis murni apapun bernilai kalklasi purna angka parameter yang pantas diboyong direbahkan peluru pelaporannya presisi ke blok pemanggilannya yang memerintah fungsinya. Disisi lain struktur susunan fungsi parameter *Hitung_Pajak*, blok barisan pemrosesannya merentang menuntut keharusan rasio di mana fungsinya wajib menyuplai menyuntik memori mereturn nilai kembali melesapkan peluru komputator jawaban akhir berupa batas perhitungan hitungan presisi nilai memori `Integer` *(int)* potongan nominal Pajak yang selesai di formulasikan oleh kalkulasi operasinya tuntas divalidasikan. 

**Q2**: Apabila anda membandingkan manipulasi limit skena utilitas modularitas mutlak rentangan asimetris penulisan "Barisan seribu baris komputator sistem panjang dicentang berderet langsung numpuk ke parameter blok mutlak dalam blok tubuh Fungsi pusat tunggal Utama `main()` doang tanpa percabangan modul samasekali" VS direfaktorisasi memecahnya di-suplai diserap memecah kedalam beranak pinak murni seratus varian pecahan prosedur `void` yang sebatas sejenis terpisah dipanggil di `main()`. Rincikan secara komputasi presisi mengapa memori referensi praktik pendelegasian pendekatan pecahan modul refaktorasi utilitas berpotongan ke dua rujukan arsitek ini disebut diagungkan diposisikan mutlak menjadi parameter batas *Program paradigm design Clean Code* mutakhir tervalidasi profesional?
**A2**: Pengkonversian paradigma perombakan rimpang ini sangat mengunggulkan esensi utilitas memori pengontrolan program batas *Reusability* (Kemampuan meng-copast memanggil parameter rimpang fungsi seratusan modul itu dipanggil secara instan mandiri digerai berulang mutlak sesuka batas utilitas perulangan kemanapun lokasinya tanpa ngetik tumpukan kode baris panjang ulang boros presisi). Sekaligus memberikan garansi kemampuan mutakhir esensi arsitekjur resiliensi *Maintainability Error Debugging Isolatif*. Andai sebuah bagian pemrosesan kalkulator pajaknya dilaporkan menubruk cacat eror *Bug* komparasinya gagal logis, programmer hanya tinggal membedah merambah dan mengisolisai mensortir operasi di sebuah spesifik kurung ruang teritori modul isolasi `hitungPajak()` terisolasi miliknya sendiri, tanpa merusak was-was kepanikan menscrolling mencari jarum jerami mencacah merusak komando relung membedah membongkar ribuan gerbong seribu baris asimetri tatanan barisan blok parameter kode murni yang berjejalan tak terbatas ruwet memanjang serampangan pada pilar batas utama `main()`.

# Algoritma Dasar: Binary Search dan Bubble Sort

**Ringkasan Materi (TL;DR)**
Menyisir presisi melintang pencari elemen presisi tersembunyi (*Searching*) dan menertibkan jajaran kordinasi blok perbatasan yang di acak urutan tatananya murni posisinya (*Sorting*) adalah utilasi instrumen mesin esensial penopang piranti komputator Data. **Bubble Sort** menertibkan data pelan-pelan murni bermodalkan menggelepar menyisir meraba berdampingan relasional menukar dua elemen tetangganya yang salah tempat, sementara **Binary Search** melacak meroket ekstrim pelacak target referensi kuncian melaju membabat rasion array sekuensial yang *wajib dipastikan absolut terutut paten sebelumnya*, lalu membelah potong parameter rimpangan komputator pencarian tersebut presisi paruh waktu merayap pangkas menjadi rentangan pembelah irisan dua potong murni tengah di setiap langkah limit penelusurannya.

**Konsep Ruang / Fisiologi / Kompleksitas Algoritma**
1. **Dinamika Bubble Sort (Pengurutan Gelembung):**
Menduduki rasional predikat perulangan batas sortir paling pelan naif elemen. Komputator iteratif barisan blok `Nested Loop` luar menungso rotasi, `Loop` iterasi dalam yang berjalan berdampingan sel langkah demi sel berjalan ke kanan membandingkan relasional berbatasan kordinasi sepasang *(Swap adjacent)* dari rimpangan pasangan sel indeks $i$ dengan indeks $i+1$. Jika sel elemen terdeteksi membelot murni rentangannya (nilai indeks parameter letak kiri ternyata keblinger memuat angka logis asimetris yang mutlak lebih Bongsor/besar nilainya mengungguli mendominasi letak tetangga indeks kanan sebelahnya), maka program memutus melempar instruksi me-*Swap* kedua balok letak memori angkanya diputar balik. Sesi putaran penuh ini mutlak menjamin rimpangan data batas nilai komputasi yang paling raksasa raksasa terekstrem bobot terbesarnya menggelepar muncul tersapu bersih terdorong menggelembung mendarat ditekankan tenggelam mentok mengendap dipaksa digusur tuntas ke dasar perbatasan indeks mutakhir letak elemen ekor ujung mentok yang berposis asimtot di batas sel kanan laci parameter rentangan array. *Kompleksitas:* $\mathcal{O}(n^2)$.
2. **Dinamika Binary Search (Pencarian Lipat Ganda Dua / Biner):**
Korelasi skema logis metode pelacak pintar tak terduga meminjam konsep limit *Divide and Conquer* utilitas. Pertama rujukan tatanan array dicek *WAJIB absolut Terurut (Sorted Array list murni)*. Pengecekan tidak ngawur disisir rentangan raba index dari sel kordinasi 0 berlarut sampai batas index akhir seperti Linear search. Kursor komputator presesi penunjuk membidik langsung melesak instan rasion menebak memotong index sel rentang paramarter yang eksak perbatasan tepat berdiri di relasional Paruh Tengah Median Array tersebut (*Median Middle Index element = Left + Right / 2*). Jikalau angka tebakan mediannya kebetulan presisi eksponensial di test mutlak masih merujuk mengunguli rasion nilai limit angka target pencarian komparasinya, Maka tatanan gerbong separuh index sekuen paruh *sebelah wilayah kanan* limit yang dipastikan murni berisikan entitas angka super raksasa lebih mutlak gede nilainya ini bakalan di delegasikan langsung dibuang dipangkas dieliminasi presisi dipotong dibuang abai terlepas sepenuhnya tuntas total separuh deretan, karena target referensi mungil pasti terisolisir mengendap dizonasi selahan separuh blok indeks irisan wilayah mutlak tertinggal sebelah rimpang array potong kirinya belaka iterasi mediannya. Begitu terus pembelahan direntang berlipat ganda reduksi presisi separuhnya kembali di batas iterasi berulang logaritmanya. *Kompleksitas Asimtot:* Sangat dahsyat meluncur kilat menukik teriris cepat membatasi $\mathcal{O}(\log n)$.

**Latihan Soal & Pembahasan (Format Q&A)**

**Q1**: Sebutkan rincian pertanggung jawaban pemicu rentangan batas skenario ekstrim asimetris di utilitas di mana pencarian *Binary Search* murni akan meledak komputasi tabrakan fungsinya macet tak bisa diserahkan dimutlak diforsir meraba digunakan secara instan di list array berikatan `[4, 2, 7, 1, 9, 3]`? 
**A1**: Metode penelusuran potong separuh lipatan utilasi *Binary Search* dipastikan rentan hancur membelot lumpuh logikanya diskenario tersebut menabrak gagal fungsional mutlak! Dikarenakan rimpangan relasi gerbong himpunan array di batasan masukan tersebut berposisi secara rentangan data mentah yang mutlak **Acak Amburadul (Unsorted murni)**. Binary search murni menyelaraskan fungsi arsitektur logikanya yang meminjam deduktif prasyarat asumsi utilitas limit paten "Tatkala komputasinya menebak median tengah di titik titik, separuh sisi letak zonasi ke ujung dijamin murni isinya angka yang membengkak lebih besar absolut komparasinya, dan separuh blok turun ke kiri referensi nilainya pasti mengkerut mengecil asimtotnya". Jika data baris array-nya berserakan acak (tidak terurut berbaris rapi selaras nilai batasannya), pencarian tebak belahan median memotong buta ini tak bisa menjangkarkan pilar logis arah potong array pencariannya. Menyasar menebak komparasi referensinya buta murni gagal merujuk arah penelusuran bloknya dipangkas lari kemana rutenya di iterasi langkah pemotongan. Rasion array data inputnya wajib diurutkan komputasi mutlak dahulu *(Pre-sorting requirement)* sebelum disuplai dilempar penelusuran rentangan eksekutor indeks target ke barisan pintu eksekutor Biner tersebut.

**Q2**: Apabila anda meneliti skenario komputator penugasan limit iteratif pengurutan rimpangan struktur array berhimpun utilitas `Bubble Sort` menertibkan jajaran blok batas berukuran 10 perbendaharaan himpunan parameter ukuran gerbong element laci murni array (ukuran barisan rentangan length rasional himpunan mutlak di batasan nilai kapasitas element tatanan n=10 presisi). Pada eksekutor limit loop terluar batas putaran rasion pengamatan langkah babak siklus eksekusi sirkulasi utuh indeks *Iterasi Luaran ke-2 batas perulangan*, ada minimal batas himpunan presisi jaminan mutlak berapa total elemen rentangan referensi berhimpun di blok zonasi memori sebelah wilayah ujung tapal pantat rimpang kanan array index yang dicalonkan nubuatan dipastikan termutasi lurus rapi eksak di relasional tempat indeks pengurutan akhir persis absolut aslinya tersebut komputasinya? Tancapkan dan rumuskan penjelasannya memadai!
**A2**: Skenario utilitas jaminan sirkulasi mesin algoritma skema gerbong komparator blok Bubble Sort menjanjikan secara niscaya rujukan presis garansi logis absolut limit terstruktur: Setiap satu persatu kelipatan putaran lari purna utuh teratas *(Utilitas one full inner passthrough)* akan menggiring memukul mengisolasi menggelembung paksa mencampakan mutlak $1$ butir laci pilar nilai elemen variabel angka paling membangkak *tergede rasio rekor mutlak tertingginya sisanya* bergeser me-rotasi tuntas ditekankan mutasi berbaris pas mentok rapi menyembul ke pelukan sel gerbong di barisan ekor paling belakang tatanan blok sekan urutan elemennya tervalidasi menempati susunan paling pojok zonasi batas tatanan di sisi index kanan. Lantaran soal referensinya mengandaikan pelacakan telah merampungkan di akhir merajut limit perputaran perhelatan blok di rasion limit iterasi lengkap ekspor putaran barisan pas luar rotasi pengamatan indeks lari siklus memutarnya babak lintasan komputasi ke-**2**, Maka divalidasi presisi absolut garansi memori relasional ada persis total terjamin utuh **$2$ butir jajaran himpunan pilar elemen (Dua elemen komparasi)** angka gerbong mutlak paling rekor rasional perbatasan paling Gede dominan super maksimal tertinggi seantero referensi array element komparasinyanya tersebut. Mereka murni terdampar merentang ter-identifkasi mutlak murni berbaris presisi mengalah di relung posisi ujung barisan pantat perbatasan penahan sisi limit gerbong element index kordinasi di pojokan wilayah kanan murni di posisi *sorted partition* rentangannya absolut menempati letaknya asalnya terurut tak tertukar.
